# GitHub Sentinel v0.0.2

## 功能特性

- 支持按订阅间隔抓取仓库事件，并基于订阅时间窗口生成报告。
- 支持手动选择开始日期和结束日期，先抓取最新事件更新事件表，再按所选日期范围生成一次 Markdown 报告。
- 支持抓取并汇总 Push 和 Issue 相关事件，Push 事件会按 commit 拆分为独立动态。
- 支持仓库事件按外部事件 ID 去重，避免重复抓取导致报告数据重复。
- 支持接入内置 LLM 能力生成项目进展简报，默认使用智谱免费系列，也可切换 Gemini 免费系列。
- LLM 报告使用固定 Markdown 简报结构，按新增功能、主要改进、修复问题分类汇总仓库动态。
- 报告名称支持按仓库名称和用户选择的日期范围生成，报告数据与对应订阅仓库绑定。
- 支持在 Dashboard 中按订阅仓库查看报告列表，并直接展示报告名称、生成时间和 Markdown 正文。
- 报告相关时间统一使用 `YYYY-MM-DD HH:mm:ss` 格式展示，报告生成时间按当前系统本地时间生成。
- 新增可提交仓库的环境变量示例文件，便于本地配置 LLM、数据库和令牌相关参数。
