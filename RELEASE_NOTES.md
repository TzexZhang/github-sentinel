# Git Sentinel v1.0.0

发布日期：2026-06-06

## 本版本新增

- 新增本地账号体系，支持注册、登录、退出登录和修改密码。
- 新增 Cookie/Token 会话机制，默认 7 天内免登录。
- 新增多用户数据绑定，仓库订阅、报告和通知任务按当前登录用户隔离。
- 新增报告管理模块，支持查看报告列表、批量勾选并物理删除报告。
- 新增用户管理模块，支持当前登录用户修改密码和退出登录。
- 新增报告列表按生成时间倒序展示，最近生成的报告显示在最上方。
- 新增企业微信完整正文通知能力，内容过长时自动拆分为多条文本消息发送。
- 新增 Notification Worker 单轮处理数量控制，默认每轮处理 50 条待发送任务。

## 本版本调整优化

- 项目名称从 `GitHub Sentinel` 调整为 `Git Sentinel`。
- 仓库订阅的通知类型调整为“不通知”“邮箱 SMTP”“企业微信通知”，移除页面上的 Webhook 选项。
- 企业微信机器人名称调整为“企业微信通知”。
- 报告生成内容优化为以业务功能和结果为导向，合并相似功能点，减少重复描述和低价值代码细节。
- 通知任务改为从报告正文读取内容，不再保留重复的通知正文快照字段。
- 所有报告数据、订阅数据和通知数据改为按当前登录用户归属和查询。
- 本地开发启动说明补充 Windows 热重载参数，降低代码修改后旧连接阻塞服务退出的概率。
